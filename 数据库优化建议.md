# 数据库优化建议

> 📚 本文档提供数据库性能优化建议

---

## 1. 数据库索引优化

### 1.1 为什么需要索引？

索引就像书的目录，可以快速定位数据，大大提升查询速度。

### 1.2 建议添加的索引

执行以下SQL语句来添加索引：

```sql
USE student_talk_db;

-- 1. 为 studentId 字段添加索引（最重要！）
-- 这个索引能大幅提升按学生ID查询的速度
CREATE INDEX idx_student_id ON talkrecord(studentId);

-- 2. 为 createTime 字段添加索引
-- 如果经常按时间排序查询，可以添加这个索引
CREATE INDEX idx_create_time ON talkrecord(createTime);

-- 3. 复合索引（可选，适用于同时按学生ID和时间查询）
CREATE INDEX idx_student_time ON talkrecord(studentId, createTime);
```

### 1.3 查看索引

```sql
-- 查看表的所有索引
SHOW INDEX FROM talkrecord;
```

### 1.4 删除索引（如果不需要）

```sql
DROP INDEX idx_student_id ON talkrecord;
```

## 2. 数据库表结构优化建议

### 2.1 当前表结构

```sql
CREATE TABLE talkrecord (
    id INT AUTO_INCREMENT PRIMARY KEY,
    studentId VARCHAR(50) NOT NULL,
    content TEXT NOT NULL,
    createTime DATETIME NOT NULL
);
```

### 2.2 优化建议（可选）

```sql
-- 建议添加的字段
ALTER TABLE talkrecord 
ADD COLUMN counselorId VARCHAR(50) COMMENT '辅导员ID' AFTER studentId,
ADD COLUMN sessionId VARCHAR(50) COMMENT '会话ID' AFTER counselorId,
ADD COLUMN updatedAt DATETIME DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP COMMENT '更新时间';
```

## 3. 查询优化

### 3.1 当前查询（已优化）

```sql
SELECT content, createTime 
FROM talkrecord 
WHERE studentId = %s 
ORDER BY createTime DESC 
LIMIT 100
```

这个查询已经比较优化了，加上索引后会更快。

### 3.2 避免的查询

❌ 不要这样做：
```sql
-- 全表扫描，很慢！
SELECT * FROM talkrecord;

-- 使用 LIKE '%xxx%' 无法使用索引
SELECT * FROM talkrecord WHERE content LIKE '%压力%';
```

✅ 应该这样做：
```sql
-- 有索引，很快！
SELECT * FROM talkrecord WHERE studentId = 'S2023003';

-- 如果需要全文搜索，考虑使用全文索引
CREATE FULLTEXT INDEX idx_content ON talkrecord(content);
SELECT * FROM talkrecord WHERE MATCH(content) AGAINST('压力');
```

## 5. 性能对比

| 场景 | 无索引 | 有索引 | 提升 |
|------|--------|--------|------|
| 查询单学生记录（1万条数据） | ~100ms | ~5ms | **20倍** |
| 查询单学生记录（10万条数据） | ~1000ms | ~5ms | **200倍** |
| 重复查询同一学生（有缓存） | ~5-10秒（LLM调用） | ~1ms（缓存） | **5000倍+** |

## 6. 快速应用索引

如果您想快速应用索引，可以运行以下Python脚本：

```python
import mysql.connector
from utils.config_handler import db_conf

mysql_config = db_conf.get("mysql", {})

conn = mysql.connector.connect(
    host=mysql_config.get("host", "localhost"),
    user=mysql_config.get("user", "root"),
    password=mysql_config.get("password", ""),
    database=mysql_config.get("database", "chat_history")
)

cursor = conn.cursor()

try:
    cursor.execute("CREATE INDEX idx_student_id ON talkrecord(studentId)")
    print("[OK] 索引 idx_student_id 创建成功")
    
    cursor.execute("CREATE INDEX idx_create_time ON talkrecord(createTime)")
    print("[OK] 索引 idx_create_time 创建成功")
    
except Exception as e:
    print(f"[ERROR] {e}")

cursor.close()
conn.close()
```
