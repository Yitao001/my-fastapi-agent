## 1. 项目概述

### 1.1 项目是什么？

这是一个**人物画像分析智能体（Agent）**，简单来说就是：
- 你给它一个学生ID
- 它去数据库里找这个学生的历史谈话记录
- 然后用AI分析这个学生的性格、遇到的问题、当前状态
- 最后生成一份详细的人物画像报告

### 1.2 项目能做什么？

✅ 从MySQL数据库读取学生谈话记录  
✅ 用AI大模型分析人物性格  
✅ 生成辅导员视角和学生视角的双视角报告  
✅ 提供API接口供其他系统调用  
✅ 支持Docker容器化部署  

### 1.3 项目的应用场景

- 高校辅导员：快速了解学生心理状态
- 心理咨询：辅助分析来访者特征
- 教育研究：批量分析学生群体画像

##### 函数2：`get_chat_history_from_db()` - 拿谈话记录

```python
def get_chat_history_from_db(participant_id: str) -> str:
    """
    作用：根据学生ID，从数据库里找出他的所有谈话记录
    就像：在文件柜里找某个学生的档案夹
    """
    # 第1步：连接数据库
    conn = get_db_connection()
    cursor = conn.cursor()
    
    # 第2步：执行SQL查询（就像说："帮我找这个学生的记录"）
    query = """
        SELECT content, created_time 
        FROM talkrecord 
        WHERE studentId = %s 
        ORDER BY created_time DESC 
        LIMIT 100
    """
    cursor.execute(query, (participant_id,))
    
    # 第3步：获取查询结果
    results = cursor.fetchall()
    
    # 第4步：把结果整理成文本
    chat_history = ""
    for row in results:
        content, talk_time = row
        chat_history += f"时间: {talk_time}\n内容: {content}\n\n"
    
    return chat_history
```

##### 函数3：`_generate_persona()` - AI分析生成

```python
def _generate_persona(chat_history: str, target_person: str, participant_id: str) -> str:
```

##### 函数4：`generate_persona_from_db()` - 主函数

```python
def generate_persona_from_db(participant_id: str) -> str:
    """
    作用：把上面几个函数串起来，一键完成
    就像：一条龙服务
    """
    # 第1步：从数据库拿谈话记录
    chat_history = get_chat_history_from_db(participant_id)
    
    # 第2步：检查有没有拿到数据
    if "未找到" in chat_history:
        return chat_history
    
    # 第3步：让AI分析
    target_person = f"ID为{participant_id}的用户"
    return _generate_persona(chat_history, target_person, participant_id)
```

## 7. 常见问题解答

### Q1: 提示 "ModuleNotFoundError: No module named 'xxx'" 怎么办？

**A:** 依赖没安装完整，重新运行：
```powershell
pip install -r requirements.txt
```

### Q2: 数据库连接失败怎么办？

**A:** 检查以下几点：
1. MySQL服务是否启动
2. `.env` 里的数据库配置是否正确
3. 用户名密码是否正确
4. 数据库是否存在

### Q3: API返回 "缺少API Key" 怎么办？

**A:** 有两个解决办法：
- **办法1**：在请求头里加上 `X-API-Key: dev_secure_key_2024_change_in_production`
- **办法2**：把 `.env` 里的 `API_KEY` 改成空的 `API_KEY=`

### Q4: 如何修改AI分析的提示词？

**A:** 编辑 `prompts/persona_prompt.txt` 文件，修改里面的内容即可。

### Q5: 如何更换AI大模型？

**A:** 修改 `model/factory.py` 文件，改成你想用的模型（如GPT-4、Claude等）。

### Q6: 向量数据库是干什么的？

**A:** ChromaDB向量数据库用来存储历史分析结果，方便以后检索相似的案例。

### Q7: 如何用Docker部署？

**A:** 项目里应该有Dockerfile，运行：
```powershell
docker build -t persona-agent .
docker run -p 8000:8000 --env-file .env persona-agent
```

### Q8: 这个项目可以商用吗？

**A:** 取决于你用的AI大模型的许可协议。代码部分可以自由使用。
