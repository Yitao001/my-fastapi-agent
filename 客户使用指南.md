## 1. 系统简介

### 1.1 什么是学生画像分析智能体？
学生画像分析智能体是一个基于人工智能的系统，它可以通过分析学生的历史谈话记录，自动生成学生的人物画像，帮助教育工作者更好地了解学生的性格、学习情况和心理状态。

### 1.2 核心特点
- ✅ **安全**：您的学生数据永远在您本地，不会上传到云端
- ✅ **简单**：只需安装一次，永久使用
- ✅ **方便**：无需公网IP，无需配置端口映射
- ✅ **智能**：基于先进的AI大模型分析
- ✅ **高效**：一次配置，开机自启，全程无感

### 1.3 系统要求
- 操作系统：Windows 7/10/11 / Linux / Mac
- Python 3.8 或更高版本（如需自己运行客户端）
- MySQL 数据库（存储学生谈话记录）

## 2. 系统架构

### 2.1 架构图

```
┌─────────────────────────────────────────────────────────────────────────┐
│                         云服务器（我们的）                              │
│  ┌──────────────────┐         ┌──────────────────┐                   │
│  │  AI智能体服务   │         │  中继服务      │                   │
│  │  (端口 8000)   │────────▶│  (端口 8002)    │                   │
│  └──────────────────┘         └────────┬─────────┘                   │
└─────────────────────────────────────┼─────────────────────────────────┘
                                      │ WebSocket
                    ┌─────────────────┴─────────────────┐
                    │                                   │
            ┌───────▼────────┐            ┌────────▼─────────┐
            │  用户1的客户端      │            │  用户2的客户端    │
            │  (您的电脑)   │            │  (其他用户)   │
            │  ┌───────────┐ │            │  ┌───────────┐│
            │  │ 本地数据库 │ │            │  │ 本地数据库 ││
            │  └───────────┘ │            │  └───────────┘│
            └────────────────┘            └──────────────────┘
```

### 2.2 工作原理
1. 您在本地运行我们提供的客户端程序
2. 客户端主动连接云服务器的中继服务
3. 当需要分析学生画像时，AI智能体通过中继服务向您的客户端发送请求
4. 您的客户端查询本地数据库，返回数据
5. AI智能体分析数据，生成学生画像

## 3. 快速开始（5分钟上手）

### 3.1 第一步：准备工作
您需要准备以下信息：
1. 您的 MySQL 数据库地址、用户名、密码
2. 数据库中存储学生谈话记录的表名和字段名
3. 我们提供的云服务器地址和中继服务地址

### 3.2 第二步：安装客户端

**Windows 用户：**
1. 解压我们提供的 `db-proxy-client.zip` 文件
2. 进入解压后的文件夹
3. 双击 `start.bat`

**Mac/Linux 用户：**
1. 解压 `db-proxy-client.zip`
2. 打开终端，进入文件夹
3. 运行：`python client.py`

### 3.3 第三步：配置客户端
1. 首次运行会自动创建 `.env` 文件
2. 用记事本打开 `.env` 文件，修改以下内容：

```env
# ==========================================
# 中继服务配置
# ==========================================
RELAY_URL=ws://47.102.145.89:8002  # 我们提供的中继服务地址
CLIENT_ID=my_school_client_001            # 您可以自己起个名字

# ==========================================
# 数据库配置
# ==========================================
MYSQL_HOST=127.0.0.1
MYSQL_PORT=3306
MYSQL_USER=root
MYSQL_PASSWORD=您的数据库密码
MYSQL_DATABASE=student_talk_db
```

### 3.4 第四步：启动客户端
保存 `.env` 文件后，再次运行 `start.bat`（Windows）或 `python client.py`（Mac/Linux）

看到以下信息表示成功：
```
============================================================
数据库代理客户端启动中...
============================================================
客户端ID: my_school_client_001
中继服务: ws://47.102.145.89:8002
数据库: 127.0.0.1:3306/student_talk_db
============================================================
正在连接中继服务: ws://47.102.145.89:8002/ws/client/my_school_client_001
已连接到中继服务: my_school_client_001
```

### 3.5 第五步：测试
联系我们，我们会帮您测试是否能正常访问您的数据库！

## 4. 详细配置步骤

### 4.1 数据库表结构要求
如果您还没有学生谈话记录表，请按以下结构创建：

**表名**：`chat_records`（或其他您喜欢的名字）

**必需字段**：
- `student_id`：学生唯一标识（VARCHAR类型）
- `content`：谈话内容（TEXT类型）
- `created_time`：创建时间（DATETIME类型）

**示例SQL**：
```sql
CREATE TABLE chat_records (
    id INT AUTO_INCREMENT PRIMARY KEY,
    student_id VARCHAR(50) NOT NULL,
    content TEXT NOT NULL,
    created_time DATETIME DEFAULT CURRENT_TIMESTAMP,
    INDEX idx_student_id (student_id)
);
```

### 4.2 配置说明
`.env` 文件中的各项配置说明：

| 配置项 | 说明 |
|--------|------|
| `RELAY_URL` | 中继服务的 WebSocket 地址，由我们提供 |
| `CLIENT_ID` | 您的客户端唯一标识，可以自己命名 |
| `MYSQL_HOST` | 数据库地址，本地数据库在您的电脑上就填 127.0.0.1 |
| `MYSQL_PORT` | 数据库端口，默认是 3306 |
| `MYSQL_USER` | 数据库用户名 |
| `MYSQL_PASSWORD` | 数据库密码 |
| `MYSQL_DATABASE` | 数据库名称 |

### 4.3 设置开机自启（可选，推荐）

**Windows：**
1. 按 `Win+R`，输入 `shell:startup`
2. 将 `start.bat` 的快捷方式复制到打开的文件夹
3. 这样每次电脑开机都会自动启动客户端

**Linux：**
1. 创建 systemd 服务文件
2. 启用并启动服务

## 5. 常见问题解答

### Q1: 客户端显示连接失败怎么办？
**A:** 请检查：
1. 网络是否正常
2. `RELAY_URL` 是否正确
3. 防火墙是否阻止了连接

### Q2: 数据库连接失败怎么办？
**A:** 请检查：
1. 数据库服务是否启动
2. 用户名密码是否正确
3. 数据库名称是否正确

### Q3: 如何确认客户端正在运行？
**A:** 看客户端窗口是否显示"已连接到中继服务"

### Q4: 可以在多台电脑上运行客户端吗？
**A:** 可以！每台电脑使用不同的 `CLIENT_ID` 即可

### Q5: 数据安全吗？
**A:** 非常安全！
- 您的数据永远在本地，不会上传到云端
- 连接由您主动发起，我们无法主动访问您的内网
- 只有您授权的请求才会查询您的数据库

### Q6: 客户端需要一直开着吗？
**A:** 如果设置了开机自启，客户端会在后台自动运行，您不需要管它
