## 2. 连接阿里云服务器

### 2.1 Windows系统连接

#### 方式1：使用PowerShell（推荐）

打开PowerShell，执行：
```powershell
ssh root@您的服务器公网IP
# 例如：ssh root@47.96.xxx.xxx
```

输入密码后即可登录。

#### 方式2：使用Xshell/Putty

1. 下载安装Xshell或Putty
2. 新建会话，输入公网IP
3. 端口：22
4. 用户名：root
5. 输入密码连接

### 2.2 Mac/Linux系统连接

打开终端，执行：
```bash
ssh root@您的服务器公网IP
```

## 5. 使用Docker部署（推荐）

Docker部署是最简单、最稳定的方式！

### 5.1 安装Docker

```bash
# 一键安装Docker
curl -fsSL https://get.docker.com | sh

# 启动Docker
systemctl start docker
systemctl enable docker

# 验证安装
docker --version
docker-compose --version
```

### 5.2 使用Docker Compose部署

#### 5.2.1 创建Docker Compose文件

在项目根目录创建 `docker-compose.yml`：

```yaml
version: '3.8'

services:
  persona-agent:
    build: .
    container_name: persona-agent
    restart: always
    ports:
      - "8000:8000"
    environment:
      # 数据库配置
      MYSQL_HOST: ${MYSQL_HOST:-localhost}
      MYSQL_USER: ${MYSQL_USER:-root}
      MYSQL_PASSWORD: ${MYSQL_PASSWORD:-}
      MYSQL_DATABASE: ${MYSQL_DATABASE:-student_talk_db}
      
      # API安全配置
      API_KEY: ${API_KEY:-}
      CORS_ORIGINS: ${CORS_ORIGINS:-*}
      RATE_LIMIT_PER_MINUTE: ${RATE_LIMIT_PER_MINUTE:-60}
      
      # 通义千问API Key（如果需要）
      DASHSCOPE_API_KEY: ${DASHSCOPE_API_KEY:-}
    volumes:
      - ./logs:/app/logs
      - ./chroma_db:/app/chroma_db
    networks:
      - persona-network

networks:
  persona-network:
    driver: bridge
```

#### 5.2.2 创建环境变量文件

在项目根目录创建 `.env.production`：

```env
# ==========================================
# 生产环境配置
# ==========================================

# 数据库配置
MYSQL_HOST=localhost
MYSQL_USER=root
MYSQL_PASSWORD=您的数据库密码
MYSQL_DATABASE=student_talk_db

# API安全配置（重要！请修改为强密码）
API_KEY=your_strong_production_api_key_change_this
CORS_ORIGINS=https://your-domain.com,http://your-domain.com
RATE_LIMIT_PER_MINUTE=60

# 通义千问API Key（如果使用通义千问）
DASHSCOPE_API_KEY=sk-xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

**重要提示**：
- ⚠️ 请将 `API_KEY` 改为复杂的随机字符串
- ⚠️ 不要将 `.env.production` 提交到Git

#### 5.2.3 创建Dockerfile

确保项目根目录有 `Dockerfile`，如果没有，创建一个：

```dockerfile
FROM python:3.10-slim

WORKDIR /app

# 安装系统依赖
RUN apt-get update && apt-get install -y \
    gcc \
    && rm -rf /var/lib/apt/lists/*

# 复制依赖文件
COPY requirements.txt .

# 安装Python依赖
RUN pip install --no-cache-dir -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple

# 复制项目代码
COPY . .

# 创建必要的目录
RUN mkdir -p logs chroma_db

# 暴露端口
EXPOSE 8000

# 启动服务
CMD ["python", "api.py"]
```

#### 5.2.4 启动服务

```bash
# 复制环境变量文件
cp .env.production .env

# 构建并启动
docker-compose up -d --build

# 查看日志
docker-compose logs -f persona-agent
```

## 6. 直接部署（不使用Docker）

如果您不想用Docker，可以直接部署。

### 6.1 安装Python 3.10+

```bash
# Ubuntu/Debian
apt install -y python3 python3-pip python3-venv

# CentOS/Alibaba Cloud Linux
yum install -y python3 python3-pip

# 验证
python3 --version
pip3 --version
```

### 6.2 创建虚拟环境

```bash
cd /root/persona-agent

# 创建虚拟环境
python3 -m venv .venv

# 激活虚拟环境
source .venv/bin/activate
```

### 6.3 安装依赖

```bash
# 使用清华镜像加速
pip install -r requirements.txt -i https://pypi.tuna.tsinghua.edu.cn/simple
```

### 6.4 配置环境变量

```bash
# 复制示例配置
cp .env.example .env

# 编辑配置
vim .env
```

按 `i` 进入编辑模式，修改配置，按 `Esc` 然后输入 `:wq` 保存退出。

### 6.5 使用systemd管理服务（生产环境推荐）

创建服务文件：

```bash
vim /etc/systemd/system/persona-agent.service
```

写入以下内容：

```ini
[Unit]
Description=Persona Analysis Agent Service
After=network.target

[Service]
Type=simple
User=root
WorkingDirectory=/root/persona-agent
Environment="PATH=/root/persona-agent/.venv/bin"
ExecStart=/root/persona-agent/.venv/bin/python api.py
Restart=always
RestartSec=5

[Install]
WantedBy=multi-user.target
```

启动服务：

```bash
# 重载systemd配置
systemctl daemon-reload

# 启动服务
systemctl start persona-agent

# 设置开机自启
systemctl enable persona-agent

# 查看状态
systemctl status persona-agent

# 查看日志
journalctl -u persona-agent -f
```

## 7. 配置Nginx反向代理（推荐）

使用Nginx作为反向代理，可以：
- 配置域名访问
- 配置HTTPS
- 负载均衡
- 静态文件缓存

### 7.1 安装Nginx

```bash
# Ubuntu/Debian
apt install -y nginx

# CentOS/Alibaba Cloud Linux
yum install -y nginx

# 启动Nginx
systemctl start nginx
systemctl enable nginx
```

### 7.2 配置Nginx

创建配置文件：

```bash
vim /etc/nginx/conf.d/persona-agent.conf
```

写入以下内容：

```nginx
server {
    listen 80;
    server_name 您的域名.com;  # 改成您的域名或服务器IP

    client_max_body_size 10M;

    location / {
        proxy_pass http://127.0.0.1:8000;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
        proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
        proxy_set_header X-Forwarded-Proto $scheme;
        
        # 超时设置
        proxy_connect_timeout 60s;
        proxy_send_timeout 300s;
        proxy_read_timeout 300s;
    }

    # 访问日志
    access_log /var/log/nginx/persona-agent-access.log;
    error_log /var/log/nginx/persona-agent-error.log;
}
```

测试并重载Nginx：

```bash
# 测试配置
nginx -t

# 重载配置
systemctl reload nginx
```

## 8. 配置防火墙和安全组

### 8.1 阿里云安全组配置

1. 登录阿里云控制台
2. 进入ECS实例 → 安全组
3. 配置入方向规则：

| 协议 | 端口 | 来源 | 说明 |
|------|------|------|------|
| TCP | 22 | 0.0.0.0/0 | SSH（建议限制为您的IP） |
| TCP | 80 | 0.0.0.0/0 | HTTP |
| TCP | 443 | 0.0.0.0/0 | HTTPS |
| TCP | 8000 | 您的IP | API（仅测试用，生产用80/443） |

### 8.2 服务器防火墙（可选）

如果服务器开启了防火墙：

```bash
# Ubuntu/Debian (ufw)
ufw allow 22/tcp
ufw allow 80/tcp
ufw allow 443/tcp
ufw enable

# CentOS/Alibaba Cloud Linux (firewalld)
firewall-cmd --permanent --add-port=22/tcp
firewall-cmd --permanent --add-port=80/tcp
firewall-cmd --permanent --add-port=443/tcp
firewall-cmd --reload
```

## 11. 常见问题

### Q1: 连接被拒绝？

**A:** 检查：
- 安全组是否开放端口
- 防火墙是否放行
- 服务是否正常运行：`systemctl status persona-agent` 或 `docker ps`

### Q2: API返回500错误？

**A:** 查看日志：
```bash
# Docker
docker-compose logs persona-agent

# systemd
journalctl -u persona-agent -n 100
```

### Q3: 如何配置HTTPS？

**A:** 使用Let's Encrypt免费证书：
```bash
apt install -y certbot python3-certbot-nginx
certbot --nginx -d 您的域名.com
```
