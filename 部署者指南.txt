第一部分：项目简介
================================================================================

1.1 项目组成
----------
项目包含三个主要部分：
- 智能体服务（api.py）：处理学生画像分析请求
- 中继服务（relay/main.py）：在智能体和客户端之间转发消息
- 客户端程序（db-proxy-client/）：用户在本地运行，连接用户的数据库

1.2 技术栈
----------
- 后端框架：FastAPI
- 大模型：支持通义千问、硅基流动、OpenAI、Anthropic
- 通信协议：WebSocket（中继服务）+ HTTP（智能体API）

第二部分：云服务器部署
================================================================================

2.1 前置要求
----------
- 阿里云ECS服务器（或其他云服务器）
- Python 3.10+
- Git
- 大模型API Key（硅基流动推荐）

2.2 第1步：克隆项目
--------------
git clone https://github.com/Yitao001/my-fastapi-agent.git
cd my-fastapi-agent

2.3 第2步：创建虚拟环境
------------------
python -m venv venv
source venv/bin/activate  # Linux/Mac

2.4 第3步：安装依赖
--------------
pip install -r requirements.txt

2.5 第4步：配置智能体
----------------
cp .env.example .env
vim .env

主要配置项：
- MODEL_PROVIDER：模型提供商（siliconflow推荐）
- SILICONFLOW_API_KEY：您的硅基流动API Key
- API_KEY：智能体服务的API Key（给用户调用时用）
- RELAY_MODE：设为true启用中继模式
- RELAY_API_URL：http://localhost:8002
- RELAY_API_KEY：中继服务的密钥
- RELAY_CLIENT_ID：与客户端配置的CLIENT_ID一致

2.6 第5步：配置中继服务
------------------
cd relay
cp .env.example .env
vim .env

配置：
- RELAY_API_KEY：设置一个安全的密钥

2.7 第6步：启动服务
--------------
启动中继服务（新终端）：
cd relay
nohup python main.py > relay.log 2>&1 &

启动智能体服务（新终端）：
cd ..
nohup python api.py > server.log 2>&1 &

查看日志：
tail -f server.log
tail -f relay/relay.log

2.8 第7步：验证部署
--------------
访问：
- 智能体API文档：http://your-server-ip:8000/docs
- 中继服务健康检查：http://your-server-ip:8002/health

第三部分：客户端打包
================================================================================

3.1 准备客户端文件
--------------
将 db-proxy-client 文件夹打包成 zip：
zip -r db-proxy-client.zip db-proxy-client/

3.2 给用户的内容
------------
用户需要收到：
1. db-proxy-client.zip
2. 客户使用指南.txt
3. 中继服务地址（ws://your-server-ip:8002）
4. 智能体API文档地址（http://your-server-ip:8000/docs）

3.3 预配置客户端（可选）
------------------
您可以先预配置好 .env.example 里的 RELAY_URL，这样用户就不用自己填了

第四部分：系统配置
================================================================================

4.1 防火墙配置
----------
确保云服务器防火墙开放以下端口：
- 8000：智能体API（HTTP）
- 8002：中继服务（HTTP + WebSocket）

4.2 安全建议
----------
1. 修改所有默认的API Key
2. 定期更换API Key
3. 使用HTTPS（生产环境）
4. 配置访问日志
5. 限制API访问频率

附录B：故障排查
================================================================================

问题1：智能体启动失败
- 检查端口是否被占用
- 检查依赖是否完整安装
- 检查 .env 配置是否正确

问题2：中继服务连接失败
- 检查防火墙是否开放8002端口
- 检查 WebSocket 连接是否被阻止
- 查看 relay.log 日志

问题3：客户端连不上中继
- 检查 RELAY_URL 是否正确
- 检查网络连接
- 检查 CLIENT_ID 是否合法
